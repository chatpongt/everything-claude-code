# เริ่มเขียนเว็บแอปจาก 0 (ฉบับ Mac + สอนแบบจับมือทำ)

คู่มือนี้สำหรับ **คนที่เพิ่งเริ่มจริงๆ** ไม่เคยเขียนโค้ดมาก่อนก็ทำตามได้
เป้าหมาย: เปิดแอปตัวอย่าง → ดัดแปลงให้เป็นของตัวเอง → clone repo คนอื่นมาแก้ได้

> 💡 เคล็ดลับ: ไม่ต้องท่องจำ ทำตามทีละขั้น ติดตรงไหนถ่ายรูปหน้าจอ/ก๊อปข้อความ error ไปถาม AI ได้เลย

---

## ตอนที่ 0 — แค่อยากเห็นแอปทำงานเดี๋ยวนี้ (30 วินาที)

ยังไม่ต้องติดตั้งอะไรทั้งนั้น:

1. เปิดโฟลเดอร์ `examples/web-app-basics/`
2. **ดับเบิลคลิกไฟล์ `index.html`**
3. มันจะเปิดในเบราว์เซอร์ (Safari/Chrome) แล้วโชว์รายการสินค้าที่ดึงมาจากอินเทอร์เน็ต ✅

แค่นี้คุณก็มี "เว็บแอปที่ดึงข้อมูลมาแสดง" ตัวแรกแล้วครับ 🎉
ลองพิมพ์คำว่า `phone` ในช่องค้นหาแล้วกด Enter ดู

## ตอนที่ 1 — เข้าใจว่าไฟล์ทำงานยังไง (อ่านครั้งเดียวพอ)

เปิดไฟล์ `index.html` ด้วยโปรแกรมแก้ข้อความ (แนะนำ **VS Code** ดูวิธีติดตั้งในตอนที่ 2)
จะเห็นว่าทั้งแอปอยู่ในไฟล์เดียว แบ่งเป็น 3 ส่วน:

| ส่วน | ภาษา | หน้าที่ | ในไฟล์อยู่ตรง |
|------|------|---------|----------------|
| โครงร่าง | **HTML** | ปุ่ม ช่องค้นหา ข้อความ | ใน `<body>` |
| หน้าตา/สี | **CSS** | สี ขนาด การจัดวาง | ใน `<style>` |
| การทำงาน | **JavaScript** | ดึงข้อมูล + แสดงผล | ใน `<script>` |

ในไฟล์ผมเขียนคอมเมนต์ภาษาไทยกำกับไว้ทุกจุดแล้ว ลองไล่อ่านดูครับ
จุดสำคัญที่สุดคือคำว่า **`fetch`** = "ไปหยิบข้อมูลจากอินเทอร์เน็ตมา"

## ตอนที่ 2 — ติดตั้งเครื่องมือบน Mac (ทำครั้งเดียว)

### 2.1 ติดตั้ง VS Code (โปรแกรมเขียนโค้ด)
1. ไปที่ https://code.visualstudio.com
2. กดปุ่มดาวน์โหลดสำหรับ Mac → เปิดไฟล์ → ลากไอคอนไปที่โฟลเดอร์ Applications
3. เปิด VS Code แล้วเปิดโฟลเดอร์โปรเจกต์ด้วยเมนู `File > Open Folder...`

### 2.2 เปิด Terminal (หน้าจอพิมพ์คำสั่ง)
- กด `Command (⌘) + Space` พิมพ์ว่า `Terminal` แล้วกด Enter
- หน้าจอดำๆ ที่ขึ้นมานี่แหละ คือที่เราพิมพ์คำสั่งสั่งงานเครื่อง

### 2.3 ติดตั้ง Homebrew (ตัวช่วยติดตั้งโปรแกรมอื่นๆ)
ก๊อปบรรทัดนี้ไปวางใน Terminal แล้วกด Enter (มันจะถามรหัสผ่านเครื่อง พิมพ์ได้เลย จอจะไม่ขึ้นตัวอักษร ปกติครับ):
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 2.4 ติดตั้ง Git และ Node.js
หลังติดตั้ง Homebrew เสร็จ พิมพ์:
```bash
brew install git node
```
ตรวจว่าติดตั้งสำเร็จ:
```bash
git --version
node --version
```
ถ้าขึ้นเลขเวอร์ชัน = สำเร็จ ✅

## ตอนที่ 3 — ดัดแปลงแอปให้เป็นของตัวเอง

ลองแก้ทีละจุดแล้วเซฟ (`⌘ + S`) จากนั้นดับเบิลคลิก `index.html` ใหม่เพื่อดูผล:

1. **เปลี่ยนหัวข้อ:** หาคำว่า `🛍️ ร้านค้าตัวอย่าง` ใน `<body>` แล้วแก้เป็นชื่อที่อยากได้
2. **เปลี่ยนสีปุ่ม:** ในส่วน `<style>` หา `background: #4f46e5;` แล้วเปลี่ยนเป็นสีอื่น เช่น `#e11d48` (แดง)
3. **เปลี่ยนข้อมูลที่ดึงมา:** ในส่วน `<script>` หาคำว่า `dummyjson.com` — นี่คือ "แหล่งข้อมูล" ลองเปลี่ยนเป็น API อื่นได้

> 🔁 วงจรการทำงานของคุณคือ: **แก้ไฟล์ → เซฟ → รีเฟรชเบราว์เซอร์ → ดูผล** ทำซ้ำไปเรื่อยๆ

## ตอนที่ 4 — clone repo จาก GitHub มาดัดแปลง (สิ่งที่คุณอยากทำ!)

นี่คือหัวใจที่คุณต้องการ ทำตามนี้:

```bash
# 1) ไปที่โฟลเดอร์ที่อยากเก็บงาน เช่น Desktop
cd ~/Desktop

# 2) ก๊อป repo ลงมา (เปลี่ยน URL เป็นของ repo ที่คุณอยากได้)
git clone https://github.com/ชื่อคน/ชื่อโปรเจกต์.git

# 3) เข้าไปในโฟลเดอร์โปรเจกต์
cd ชื่อโปรเจกต์

# 4) ดูว่าข้างในมีไฟล์อะไรบ้าง
ls
```

**แล้วรันยังไง?** ดูในไฟล์ `README.md` ของ repo นั้นเสมอ ส่วนใหญ่จะเป็นแบบใดแบบหนึ่ง:
- เปิดไฟล์ `index.html` ตรงๆ (แบบแอปตัวอย่างนี้) — ง่ายสุด
- หรือเป็นโปรเจกต์ที่ต้องสั่ง:
  ```bash
  npm install     # ติดตั้งสิ่งที่โปรเจกต์ต้องใช้ (ทำครั้งแรกครั้งเดียว)
  npm run dev     # เปิดแอปขึ้นมา แล้วเอา URL ที่ขึ้น (เช่น http://localhost:3000) ไปเปิดในเบราว์เซอร์
  ```

> ⚠️ ถ้าเจอ error อย่าเพิ่งตกใจ — ก๊อปข้อความ error ทั้งหมดไปถาม AI ได้เลย บอกว่า "รันแล้วเจอ error นี้ แก้ยังไง" มันจะช่วยไล่ให้

## ตอนที่ 5 — ใช้ AI (Claude) เป็นผู้ช่วยส่วนตัว

จุดแข็งของคุณคือ copy-paste — ใช้ให้เป็นประโยชน์! เวลาจะแก้อะไร ลองสั่ง AI แบบนี้:

- "ช่วยเพิ่มปุ่มสำหรับ... ในไฟล์ index.html ให้หน่อย"
- "โค้ดส่วนนี้ทำอะไร อธิบายเป็นภาษาไทยแบบง่ายๆ" (แล้ววางโค้ดไป)
- "ผมอยากเปลี่ยนจากดึงสินค้า เป็นดึงข่าว ต้องแก้ตรงไหน"
- "รันแล้วเจอ error นี้ [วาง error] แก้ยังไง"

**เคล็ดลับการถาม AI ให้ได้คำตอบดี:** บอก 3 อย่างเสมอ → (1) อยากได้อะไร (2) ตอนนี้เป็นยังไง (3) ติดตรงไหน

## ตอนที่ 6 — แหล่งข้อมูลฟรีให้ลองดึงมาเล่น (ไม่ต้องสมัคร ไม่ต้องใช้คีย์)

เอา URL พวกนี้ไปแทนที่ `dummyjson.com` ในโค้ดเพื่อลองเปลี่ยนข้อมูล:

| อยากได้ | URL |
|---------|-----|
| สินค้า (รูป+ราคา) | `https://dummyjson.com/products` |
| รายชื่อผู้ใช้ปลอม | `https://dummyjson.com/users` |
| โพสต์/บทความปลอม | `https://jsonplaceholder.typicode.com/posts` |
| เกร็ดความรู้แมว | `https://catfact.ninja/facts` |

> หมายเหตุ: แต่ละ API คืนข้อมูลหน้าตาต่างกัน เวลาเปลี่ยนแหล่ง อาจต้องแก้ชื่อ field (เช่น `p.title`, `p.price`) ให้ตรงด้วย ลองถาม AI ช่วยปรับได้

## สรุปคำสั่งที่ใช้บ่อย (เก็บไว้เปิดดู)

```bash
git clone <url>     # ก๊อป repo จาก GitHub มาที่เครื่อง
cd <โฟลเดอร์>        # เข้าไปในโฟลเดอร์
ls                  # ดูไฟล์ในโฟลเดอร์ปัจจุบัน
npm install         # ติดตั้งสิ่งที่โปรเจกต์ต้องใช้
npm run dev         # เปิดรันแอป (โปรเจกต์ที่มี build step)
```

ขอให้สนุกกับการเขียนโค้ดครับ เริ่มจาก 0 วันนี้ อีกไม่นานคุณจะดัดแปลง repo ได้เองสบายๆ 💪
